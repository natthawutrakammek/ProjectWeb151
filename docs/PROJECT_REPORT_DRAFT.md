# โครงร่างรายงานโครงงาน

## ชื่อโครงการ
DormEase หอพัก - ระบบจัดการหอพักสำหรับเจ้าของหอและลูกหอ

## วัตถุประสงค์ของระบบ
- ช่วยเจ้าของหอจัดการห้องพัก ค่าเช่า ค่าน้ำ ค่าไฟ ค่าใช้จ่ายอื่น และบิลรายเดือน
- ช่วยลูกหอตรวจสอบยอดชำระและประวัติบิลของตนเอง
- ลดการคำนวณซ้ำและลดความผิดพลาดจากการบันทึกด้วยมือ

## ผู้ใช้งานเป้าหมาย
- เจ้าของหอ/ผู้ดูแลหอพัก
- ลูกหอ/ผู้เช่าห้องพัก

## Functional Requirements
- ระบบยืนยันตัวตน Login / Logout / Register
- ระบบกำหนดสิทธิ์ owner และ tenant
- เจ้าของหอ CRUD ห้องพัก
- เจ้าของหอ CRUD ค่าใช้จ่ายอื่น
- เจ้าของหอบันทึกและแก้ไขบิลรายเดือน
- ลูกหอดูบิลล่าสุดและประวัติบิลย้อนหลัง
- ค้นหาและกรองข้อมูล
- แชทระหว่างเจ้าของหอกับลูกหอ

## การออกแบบฐานข้อมูลโดยสรุป
- users: เก็บบัญชีผู้ใช้ บทบาท และห้องที่ลูกหอผูกอยู่
- rooms: เก็บข้อมูลห้อง ค่าเช่า สถานะ และหมายเหตุ
- settings: เก็บชื่อหอพัก ราคาน้ำไฟ ขั้นต่ำ ช่องทางติดต่อ และรหัสลงทะเบียนที่ hash แล้ว
- charges: เก็บค่าใช้จ่ายอื่นที่นำไปคิดในบิลใหม่
- bills: เก็บบิลรายเดือนของแต่ละห้อง
- bill_charges: เก็บ snapshot ค่าใช้จ่ายอื่นในบิลแต่ละใบ
- messages: เก็บข้อความแชท
- sessions: เก็บ session สำหรับ login
- activity_logs: เก็บกิจกรรมเพื่อช่วยตรวจสอบการใช้งาน

## Security และ UX
- Hash password ด้วย scrypt
- ใช้ prepared statement เพื่อลด SQL injection
- Escape output เพื่อลด XSS
- ตรวจ input จาก form
- แยกเมนูและ route ตามสิทธิ์ผู้ใช้
- มีข้อความ error/success ที่อ่านเข้าใจง่าย
- Layout รองรับหน้าจอ desktop และ mobile

## แนวทางพัฒนาต่อ
- เพิ่ม upload สลิปโอนเงิน
- เพิ่ม export รายงานเป็น CSV/PDF
- เพิ่ม notification เมื่อบิลใหม่ถูกบันทึก
- เพิ่ม dashboard chart รายรับรายเดือน
